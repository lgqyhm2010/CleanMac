# CleanMac

**Нативный, приватный, AI-ассистированный очиститель диска для macOS.**

Ищите кэши, логи, дубликаты, крупные файлы и неиспользуемые приложения, проверяйте обезличенные метаданные с помощью установленного AI CLI и безопасно перемещайте их в Корзину. CleanMac не требует учётной записи и не содержит телеметрии; AI CLI может использовать сетевой сервис своего провайдера.

## Что такое CleanMac?

CleanMac — это нативное приложение для macOS (SwiftUI), которое **безопасно** освобождает место на диске. Оно сканирует ваш Mac в поисках кандидатов на очистку — кэшей, логов, временных файлов, дубликатов, слишком крупных файлов, оставшихся данных приложений — присваивает каждому уровень риска и позволяет всё проверить, прежде чем что-либо будет затронуто. Файлы отправляются в **Корзину**, а не удаляются сразу навсегда.

Что делает его особенным: он может передать **ограниченные обезличенные метаданные** выбранных объектов установленному AI CLI (Claude Code, Codex или Gemini CLI), чтобы получить второе мнение. Ваш вопрос отправляется без изменений; CleanMac не добавляет содержимое файлов или полные пути, а CLI может обращаться к настроенному провайдеру.

## Возможности

- 🧹 **Умное сканирование** — кэши, логи, временные файлы, Корзина, мусор разработчика (производные данные Xcode), загрузки и неклассифицированные «прочие» файлы.
- 📦 **Поиск дубликатов** — группирует файлы по хешу содержимого SHA-256 и сохраняет самую свежую копию.
- 🐘 **Крупные файлы** — показывает файлы больше настраиваемого порога (по умолчанию 500 МБ).
- 🗑️ **Удаление приложений** — перемещает выбранные приложения в Корзину; пользовательские данные поддержки остаются нетронутыми, чтобы избежать случайной потери.
- 🤖 **AI-проверка (локальная)** — попросите установленный AI CLI разбить кандидатов на *безопасно удалить*, *рискованно* и *требует проверки*.
- 🛡️ **Безопасность прежде всего** — более 20 правил безопасности и три уровня защиты блокируют удаление системных данных, данных Mail/Messages/Safari и Keychain.
- 🌍 **11 языков** — полностью локализованный интерфейс, который следует за языком вашей системы или заданному вручную.
- 🔒 **Явная граница конфиденциальности** — без телеметрии и учётной записи CleanMac; передача в AI обезличена и раскрыта.

## AI-проверка (локальная)

CleanMac обнаруживает поддерживаемые AI CLI в вашем `PATH` и позволяет проверить до 80 кандидатов. Он отправляет ваш вопрос и структурированные анонимные метаданные (ID, размер, дату, категорию, риск и ID правил), запускает CLI из уникального пустого временного каталога с тайм-аутом 120 секунд и принимает ответ только тогда, когда каждый объект классифицирован ровно один раз.

| Инструмент | Бинарный файл | Модели на выбор |
|------|--------|---------------------|
| **Claude Code** | `claude` | Default · Fable · Opus · Sonnet · Haiku |
| **Codex** | `codex` | Default · gpt-5.5 · gpt-5.4 · gpt-5.4-mini |
| **Gemini CLI** | `gemini` | Default · Pro · Flash |

Список кандидатов и ваш запрос передаются в CLI через stdin/аргументы как подпроцесс — **они никогда не покидают вашу машину**. CleanMac также удаляет собственные маркеры сессии из окружения дочернего процесса, чтобы CLI не мог ошибочно определить вложенную сессию.

## Приватность и безопасность

- **Раскрытие сети AI.** CleanMac не содержит телеметрии, но установленный AI CLI может обращаться к провайдеру. Проверка отправляет ваш вопрос без изменений, анонимные ID и ограниченные метаданные, но не содержимое файлов и не автоматически собранные полные пути.
- **Корзина, а не `rm`.** Всё перемещается через `FileManager.trashItem(at:)`, поэтому вы можете восстановить.
- **Уровни защиты.** `allowed` (кэши/логи/временные) → `requiresReview` (исходный код, облачное хранилище, загрузки, данные разработки) → `blocked` (корень системы, данные приложений, Mail/Messages/Safari, данные браузера, Keychain).
- **Full Disk Access** необязателен, но рекомендуется, чтобы сканирование видело защищённые расположения Library. CleanMac проведёт вас через его предоставление в Системных настройках.

## Установка

### Загрузка (рекомендуется)

1. Возьмите последний **`CleanMac.dmg`** со [страницы релизов](https://github.com/lgqyhm2010/CleanMac/releases/latest).
2. Откройте DMG и перетащите **CleanMac** в **Applications**.

> Официальные загрузки подписаны и нотаризованы. Если Gatekeeper не может проверить файл, не отключайте карантин; удалите его и скачайте заново с официальной страницы Releases.

### Требования

- macOS **14.0 (Sonoma)** или новее
- Apple Silicon или Intel

## Сборка из исходников

```bash
git clone https://github.com/lgqyhm2010/CleanMac.git
cd CleanMac

# Собрать и запустить бандл приложения (создаёт dist/CleanMac.app и запускает его)
./script/build_and_run.sh

# Или просто собрать с помощью SwiftPM
swift build --product CleanMac

# Запустить тесты
swift test
```

**Тулчейн:** Swift 6.0 (Xcode 16+). Пакет предоставляет исполняемую цель `CleanMac` (приложение) и библиотечную цель `CleanMacCore` (логика, модели, сервисы — вынесены отдельно для тестируемости).

## Упаковка DMG

Единый скрипт собирает приложение и упаковывает распространяемый DMG с установкой перетаскиванием:

```bash
# Локальный непубликуемый preview
./script/build_dmg.sh --unsigned

# Официальный релиз; нужны Developer ID и данные нотаризации
CLEANMAC_VERSION=1.0.0 CLEANMAC_BUILD_NUMBER=1 \
  NOTARY_PROFILE=CleanMacNotary ./script/build_dmg.sh --release
```

Официальный релиз работает fail-closed: ошибка данных, версии, архитектуры, подписи, нотаризации, stapling или Gatekeeper останавливает сборку. Настройка идёт через переменные окружения; секреты не зашиваются в код:

| Переменная | Назначение |
|----------|---------|
| `CODESIGN_IDENTITY` | Идентичность для подписи, например `Developer ID Application: Name (TEAMID)`. Определяется автоматически, если не задано. |
| `CLEANMAC_VERSION` / `CLEANMAC_BUILD_NUMBER` | Версия релиза и числовой номер сборки. |
| `NOTARY_PROFILE` | Имя профиля keychain для `notarytool` (см. [`docs/RELEASING.md`](docs/RELEASING.md)). |
| `APPLE_ID` / `APPLE_TEAM_ID` / `APPLE_APP_PASSWORD` | Альтернативные учётные данные для нотаризации (используются в CI). |

> `--unsigned` предназначен только для локальной/PR-проверки и не публикуется как релиз. См. [`docs/RELEASING.md`](docs/RELEASING.md).

PR и push в `main` создают только read-only unsigned preview. Публикация возможна лишь для тега `v*` после всех проверок ([workflow](.github/workflows/release-dmg.yml)).

## Локализация

CleanMac поставляется на **11 языках**: английском, упрощённом китайском, традиционном китайском, японском, испанском, французском, арабском, хинди, португальском (Бразилия), русском и бенгальском. Интерфейс по умолчанию следует за языком вашей системы и может быть переопределён в **Настройках**.

Строки находятся в `Sources/CleanMacCore/Resources/<locale>.lproj/Localizable.strings` за абстракцией `L10n`. Чтобы добавить язык, добавьте новую папку `.lproj` с переведённым `Localizable.strings`, добавьте case в `AppLanguage` и пересоберите.

## Структура проекта

```
Sources/
  CleanMac/          Цель приложения — SwiftUI-представления, хранилища, оболочка AppKit, меню
  CleanMacCore/      Библиотечная цель — модели, сервисы, правила безопасности, локализация
Tests/
  CleanMacCoreTests/ Тесты основной логики
  CleanMacUITests/   Тесты приложения/хранилищ
script/
  build_and_run.sh   Собрать бандл .app и запустить его
  build_dmg.sh       Собрать + упаковать (и подписать/нотаризовать) DMG
```
